## 슬라이드 04-A 용어집 (★ 비전공자 5초 인입)

이 한 페이지만 보시면 뒤 모든 숫자가 이해됩니다. 8개만 짧게요. ① **3계층** = 약국·하수·검색. ② **시간 순 교차검증 (walk-forward)** = AI가 미래 못 보게 시간 순으로만 학습·평가. ③ **F1·Precision·Recall·FAR** — Precision은 경보 낸 거 중 진짜 비율, Recall은 진짜 중 잡은 비율, F1은 둘 합친 종합, FAR는 잘못 낸 경보 비율. ④ **종합 위험도 composite** = 3계층 가중평균 0~100, 30/55/75 넘으면 노랑/주황/빨강. ⑤ **RAG** = AI가 답하기 전에 가이드 문서 먼저 찾아 인용. ⑥ **SSE** = 글자 한 자씩 실시간 전송. ⑦ **Granger 인과검정 / CCF** = A가 B보다 먼저 움직이는지 통계 검증. ⑧ **B2G·PoC** = 정부 납품 / 시범 운영.

## 슬라이드 05 시스템 구조

흐름만 기억해주세요 — 수집 → 저장 → 분석 → 표시. AI 블록 셋: TFT는 시계열 예측, Autoencoder는 평소와 다른 패턴 탐지, RAG+Claude는 근거 문서 인용해서 리포트. Kafka는 운영 단계 예정이라 지금은 DB 직접 INSERT (점선).

## 슬라이드 06 캡스톤 결정사항

8주 동안 직접 결정한 것들 — KOWAS 픽셀 분석 (OCR 거부), DB 직접 INSERT (Kafka 보류), Claude 채택 (한국어+비용), walk-forward 검증 (gap=4), 약국 데이터는 네이버 대체 지표 (약사회 미접근). 표 한 줄 한 줄이 결정 근거입니다.

## 슬라이드 06-A 데이터셋·라벨 (★ 교수 1순위 질문 차단)

여기서 교수님이 즉시 찌를 사항을 정리합니다. ① **데이터셋 크기**: layer_signals 3,228건 (OTC 1,074 / KOWAS 1,054 / 검색 1,100), KCDC ground truth 1,020건. ② **양성 라벨 정의**: KCDC ILI 발생률이 직전 4주 baseline 대비 +30% 이상 상승 = y=1. **양성 113건 (11.1%) · 음성 907건 (88.9%)** — class imbalance 명시. ③ **두 가지 측정값 분리**: synthetic_hardened F1 0.667 (모델 자체 능력) vs 17 region real F1 0.621 (일반화 능력). 두 값 절대 평균/혼동 안 합니다. ④ **walk-forward 분할**: TimeSeriesSplit n_splits=5, gap=4 — 56주를 시간 순 5등분, 학습/검증 사이 4주 공백. 무작위 분할은 미래→과거 누설로 0.85 같은 거짓 점수 만들어서 시계열에선 금지.

## 슬라이드 08 코드 깊이 ① 파이프라인

여기가 가장 자랑하고 싶은 코드입니다. KOWAS 주간보고 PDF는 차트 그림만 있고 raw 숫자가 없어요. OCR은 글자 인식 오류로 깨지죠. 그래서 OCR 없이 차트 막대 색깔을 픽셀 단위로 검사. 코로나 파랑·인플루엔자 주황·노로 노랑. RGB 마스크로 막대 분리 → 높이 비율로 0~100 정규화 (KDCA 정의와 동일). 수동 0건 → 자동 952건. 같은 PDF 주면 누가 돌려도 같은 숫자.

## 슬라이드 10 데모

## 슬라이드 11 실측 성능 (★ 두 측정값 분리)

핵심 숫자입니다. **두 가지 평가 세트, 절대 섞지 않습니다.** ① synthetic_hardened (XGBoost walk-forward, gap=4): F1 0.667, Precision 1.000, AUC 0.931, Recall 0.500, 오경보 0건. ② 17 region real backtest (KCDC 1,020건): F1 0.621, Recall 0.838, FAR 0.557, **평균 +5.9주 선행**. Recall이 0.500 vs 0.838 다른 이유는 다른 평가 세트라서. + 부호는 KCDC peak보다 N주 빠르다는 뜻. Granger 실측: L3 검색→임상 p=0.007 (유의), composite p=0.021 (유의).

## 슬라이드 11-A 모델·하이퍼·XAI (★ 교수 모델 선정 질문 차단)

여기서 모델 선정 근거를 정리합니다. ① **모델 비교 5종**: ARIMA F1 0.412, Prophet 0.503, RandomForest 0.598, **XGBoost 0.667 (채택)**, LSTM 0.640. LSTM은 56주차 데이터로는 과적합. ② **앙상블 가중치 ablation 4종**: 균등 0.589 → L2 강조 0.605 → **0.35:0.40:0.25 채택 (F1 0.621, FAR 0.557 최적)** → 0.40:0.40:0.20 차이 미미. ③ **하이퍼파라미터** (model_config.yaml): XGBoost learning_rate 0.05, max_depth 4 (grid search 결과). TFT hidden_size 64, attention_heads 4, prediction_lengths [7,14,21]일. ④ **XAI · TFT 변수 중요도**: confirmed_future_center 0.269, **l2_wastewater 0.224 (2위)**, l3_search 0.093, l1_otc 0.086. L2 하수가 모델에서도 압도적 2위 = 사람 직관과 일치.

## 슬라이드 12 코드 깊이 ② 검증

walk-forward 코드 자체입니다. TimeSeriesSplit(n_splits=5, gap=4)으로 시간 순서 보존, 학습/검증 사이 4주 간격. 거기서 F1 0.667이 나옵니다. 시계열을 무작위로 섞으면 미래가 학습에 새어 들어와 거짓 점수가 됩니다. 이 정직함이 정부 납품의 전제입니다.

## 슬라이드 12-A 17지역 일반화 (★ 일반화 + Gate A 폐기 + 타임라인)

3가지를 한 슬라이드에 — ① 17개 시·도 막대 차트 + 4지역 v1 비교 표 (Recall 0.906→0.838, F1 0.661→0.621, 평균 거의 동일 = 4지역이 우연 과적합 아님). ② 의사결정 정직성: FAR 추가 안전장치(Gate A)를 임계값 4종(25/15/10/5) sweep → 0/4 후보가 동시 충족 못 함 → 솔직히 폐기. Gate B(2계층 교차검증)만 유지. ③ 17지역 composite 시계열 (6×3): 색칠 음영=경보 발령 기간, 빨간 막대=KCDC 신고 증가. 음영이 항상 빨간 막대보다 왼쪽 = 우리가 먼저 잡았다.

## 부록 — 자주 받는 질문 (예상 Q&A)

**Q1. F1 0.667 낮지 않나요?**
시간 순 교차검증으로 미래 가린 채 측정한 엄격값입니다. 무작위 분할 0.85와 직접 비교 금지. **Precision 1.000, 오경보 0건이 정부 납품엔 더 중요한 지표**.

**Q2. F1 0.667과 0.621은 왜 둘 다 있나요?**
완전히 다른 평가 세트. 0.667은 합성 hardened (모델 자체 능력), 0.621은 17지역 실데이터 (일반화 능력). 슬라이드 06-A에 분리 표 있습니다. 평균/혼동 절대 안 합니다.

**Q3. 양성 비율이 11%면 Precision 1.000이 자명한 거 아닌가요?**
class imbalance 영향 큰 게 맞습니다. 그래서 Precision 만 쓰지 않고 F1·Recall·FAR을 같이 봅니다. Recall 0.500은 진짜 절반을 놓쳤다는 솔직한 한계 — 보수적 운영 결과입니다.

**Q4. XGBoost 왜 채택?**
ARIMA / Prophet / RandomForest / XGBoost / LSTM 5종 walk-forward 비교에서 F1 최고 + Precision 1.000 안정성 (슬라이드 11-A). LSTM은 56주차 데이터로 과적합.

**Q5. 가중치 0.35:0.40:0.25 어떻게 결정?**
4가지 비율 ablation: 균등 0.589 → L2 강조 0.605 → 0.35:0.40:0.25 (0.621, 채택) → 0.40:0.40:0.20 차이 미미. L2 하수가 가장 빠른 선행지표라 최고 가중치.

**Q6. TFT는 분류에 안 쓴다면 왜 있나요?**
TFT는 회귀(미래 환자수 예측) + XAI(변수 중요도) 용. 슬라이드 11-A에 변수 중요도 5위 — l2_wastewater 0.224가 2위로 나와 사람 직관과 일치. best_val_loss 1.88.

**Q7. Autoencoder는 어디서 쓰나요?**
이상탐지(unsupervised novelty detection) 용. 5번 시스템 구조에 있고 백엔드 endpoint `/api/v1/predictions/anomaly` 는 stub 상태로 솔직히 명시 — 학습은 Phase 3.

**Q8. 검색 데이터는 구글이 실패한 거 아닌가요?**
정확합니다. 그래서 검색 단독 사용 금지, 약국·하수 중 하나가 같이 안 오르면 경보 안 울림 (슬라이드 04 교차검증).

**Q9. 시·군·구 확장 가능?**
HIRA OpenAPI 가 시·군·구 단위 의약품 청구 무료 제공 (1~2주). 우리 코드는 region 컬럼만 바꾸면 작동. PoC는 9월 로드맵.

**Q10. AI 환각 막는 법?**
RAG — Claude가 답하기 전 WHO·KDCA 가이드 문서 먼저 찾아 인용. PDF 리포트 마지막에 출처 5건 자동 박힘.

**Q11. 6주 안에 진짜 시·군·구 PoC 가능?**
6주 목표는 코드 적재까지. 지자체 시범 운영(PoC)은 9월.

**Q12. 코로나·노로도 되나요?**
신호 수집은 됨 (KOWAS 자동 다운로더가 3종 모두 추출). ground truth가 인플루엔자만이라 모델 학습은 인플루엔자만. KCDC API 확장 시 즉시 학습 가능 아키텍처.

**Q13. 발표용 데모 데이터 vs 실제?**
대시보드는 모두 TimescaleDB 실데이터 (3,228 row). 시연일이 인플루엔자 종료기라 전국 GREEN. 슬라이드 10에서 2025-12-14 시점 시간 되감기로 부산 RED · 대전 ORANGE 시연.

**Q14. SCI 논문 어떤 주제?**
"Multi-signal cross-validated influenza early warning with public-data alternatives in Korea" — 의료정보학 저널 (JAMIA/JBI) 또는 공공보건 저널 타깃. 6개월 목표.

**Q15. 합성 데이터는 어떻게 만들었나요?**
실데이터 분포에서 noise + drift 인위 주입한 hardened 합성 (synthetic_hardened). 양성 비율·계절성은 KCDC 분포 따라가도록 설계. 무작위 합성이 아니라 **모델 능력 측정 전용 hardcase 데이터**.
