1. 이 프로젝트가 뭔지 한 줄로 설명하면?
═══════════════════════════════════════════════════

  "감기·독감이 크게 퍼지기 1~3주 전에 미리 알려주는 조기경보 시스템"


  ▶ 비유: 우리 몸의 면역 시스템처럼

    우리 몸이 감기 바이러스를 만나면,
    열이 나거나 콧물이 나기 전에 이미 면역세포가 싸우기 시작합니다.
    병원에 갈 때는 이미 감기가 한창인 거죠.

    이 시스템은 도시 전체를 하나의 몸으로 보고,
    "시민들이 감기약을 많이 사기 시작했다",
    "하수도에서 바이러스가 검출됐다",
    "검색창에 '독감 증상'을 많이 검색한다"
    같은 신호를 먼저 포착해서 병원이 터지기 전에 경보를 울립니다.

    → 이름도 그래서 "Urban Immune System (도시 면역 시스템)"입니다.

2. 왜 이게 필요한가요?
═══════════════════════════════════════════════════

  지금 문제:
  ┌─────────────────────────────────────────────────┐
  │ 감염병이 퍼지고 있어도, 환자가 병원에 가야     │
  │ 데이터가 생깁니다. 질병청 보고서는 1~2주 뒤에  │
  │ 나와요. 이미 한창 퍼진 다음에 알게 되는 거죠.  │
  └─────────────────────────────────────────────────┘

  우리 해결책:
  ┌─────────────────────────────────────────────────┐
  │ 병원 가기 전에 나타나는 신호 3가지를 동시에    │
  │ 감시해서, 1~3주 먼저 경보를 줍니다.            │
  └─────────────────────────────────────────────────┘

  ▶ 비유: 비 오기 전 날씨 예보

    기상청은 비가 오고 나서 "비가 왔습니다"라고 말하지 않아요.
    구름, 기압, 온도 등 여러 신호를 분석해서 미리 예보합니다.
    우리 시스템도 똑같이 3가지 신호로 감염병을 예보합니다.

3. 3가지 신호 (Layer) 설명
═══════════════════════════════════════════════════

  ┌──────────────────────────────────────────────────────────┐
  │ Layer 1 — 약국 OTC (Over-the-Counter, 처방전 없이 사는 약) │
  │                                          [1~2주 먼저 탐지] │
  ├──────────────────────────────────────────────────────────┤
  │                                                          │
  │  비유: "편의점에서 우산이 갑자기 많이 팔리면 비가 온다"  │
  │                                                          │
  │  사람들이 아프기 시작하면 제일 먼저 하는 행동이          │
  │  약국에 가서 감기약·해열제를 사는 겁니다.               │
  │  이 구매 데이터를 네이버 쇼핑인사이트 API로 수집합니다. │
  │                                                          │
  │  모니터링 품목: 감기약, 해열제, 종합감기약, 타이레놀 등  │
  └──────────────────────────────────────────────────────────┘

  ┌──────────────────────────────────────────────────────────┐
  │ Layer 2 — 하수도 바이오마커                              │
  │                                          [2~3주 먼저 탐지] │
  ├──────────────────────────────────────────────────────────┤
  │                                                          │
  │  비유: "아파트 하수구에 뭐가 흘러내려 오는지 보면       │
  │         몇 층에서 무슨 음식을 해먹었는지 알 수 있다"    │
  │                                                          │
  │  바이러스에 감염된 사람의 소변·대변에는 바이러스가       │
  │  들어있습니다. 이게 하수도로 흘러가서 하수처리장에       │
  │  쌓이죠. 하수처리장 샘플을 분석하면 그 지역에           │
  │  감염자가 몇 명인지 증상 전에도 알 수 있습니다.         │
  │                                                          │
  │  데이터 출처: KOWAS (한국 하수도 감시 시스템) PDF 공개자료│
  └──────────────────────────────────────────────────────────┘

  ┌──────────────────────────────────────────────────────────┐
  │ Layer 3 — 검색어 트렌드                                  │
  │                                          [1~2주 먼저 탐지] │
  ├──────────────────────────────────────────────────────────┤
  │                                                          │
  │  비유: "친구들 단톡방에서 '머리가 아프다'는 말이         │
  │         갑자기 늘면 뭔가 유행하는 거 아닐까?"           │
  │                                                          │
  │  사람들이 아프기 시작하면 네이버에 "독감 증상",          │
  │  "고열 원인", "몸살 원인" 같은 검색을 많이 합니다.     │
  │  이 검색 트렌드를 네이버 DataLab API로 수집합니다.      │
  │                                                          │
  │  ⚠️ 주의: 이 신호만 단독으로 쓰면 안 됩니다.           │
  │     구글 독감 예측(2013년)이 검색만 믿었다가            │
  │     실제보다 2배나 과대예측해서 폐지된 사례가 있어요.   │
  │     반드시 Layer 1, 2와 같이 써야 합니다.               │
  └──────────────────────────────────────────────────────────┘

  ★ 핵심 규칙: 3개 중 2개 이상이 동시에 올라야 경보 발령!
    (하나만 올라도 경보 안 냄 — 오경보 방지)

4. 경보 단계 설명
═══════════════════════════════════════════════════

  ┌───────────┬────────────────────────────────────────────┐
  │  🟢 GREEN  │ 안전. 걱정 없음.                           │
  │  🟡 YELLOW │ 주의. 손 씻기, 마스크 챙겨두기             │
  │  🟠 ORANGE │ 경고. 노약자 외출 자제, 의료기관 준비      │
  │  🔴 RED    │ 위험. 대규모 유행 임박, 긴급 조치 필요     │
  └───────────┴────────────────────────────────────────────┘

  종합 점수 계산 방식:
  → 종합점수 = (약국구매 × 35%) + (하수도 × 40%) + (검색트렌드 × 25%)
  → 하수도에 가장 높은 비중: 바이러스가 있으면 100% 나오는 객관적 증거라서

5. 시스템이 어떻게 돌아가나? (전체 흐름)
═══════════════════════════════════════════════════

  [데이터 수집] → [저장] → [분석·예측] → [경보] → [화면에 표시]

  ① 매주 자동으로 약국·하수도·검색 데이터 수집
       (로봇이 알아서 API 호출하고 PDF 파일 다운로드)

  ② TimescaleDB (시간 특화 데이터베이스)에 저장
       (비유: 날짜·시간 순으로 정리된 아주 큰 스프레드시트)

  ③ AI 모델 3개가 분석
       - XGBoost: "앞으로 2주 뒤에 독감 유행할 확률"
       - Autoencoder: "지금 수치가 평소랑 너무 다른 이상한 상황인가?"
       - RAG-LLM (Claude AI): "상황을 사람이 읽기 쉽게 리포트로 작성"

  ④ 경보 레벨 결정 후 대시보드에 표시
       - 지도에 17개 시·도별로 색깔로 표시
       - AI가 한국어로 현황 리포트 자동 생성

  ⑤ 보건 당국과 시민에게 맞춤 권고사항 제공

7. 실제 성능은 어떻게 나왔나요?
═══════════════════════════════════════════════════

  (2024~2025 독감 시즌 52주 데이터 기준 검증)

  ┌──────────────────────────────────────────────────────┐
  │  F1 점수: 0.667                                      │
  │  → 경보를 울려야 할 때 울린 비율과 정확도의 종합점수 │
  │  → 100점 만점에 67점 (목표 70점, 거의 달성)          │
  │                                                      │
  │  Precision (정밀도): 1.00                            │
  │  → 경보를 울렸는데 실제로 유행한 비율 = 100%        │
  │  → 오경보 0건! 경보 울리면 진짜 유행한다는 뜻        │
  │                                                      │
  │  AUC-ROC: 0.93                                       │
  │  → 1.0이 만점, 0.93은 매우 우수한 수준               │
  │                                                      │
  │  선행 탐지 리드타임: 평균 5.9주 선행                 │
  │  → 실제 확진자가 폭발하기 약 6주 전에 미리 경보      │
  └──────────────────────────────────────────────────────┘

  ▶ 비유로 설명하면:

    야구에서 타자가 배트를 휘두르기 전에 투수가 공을 던지는 방향을
    미리 예측하는 것처럼, 우리 시스템은 독감이 폭발하기
    6주 전에 "이제 슬슬 조심해야 할 것 같다"고 알려줍니다.
    그리고 경보를 울린 경우 100%가 실제 유행이었습니다.

9. Q&A 예상 질문 답변
═══════════════════════════════════════════════════

  Q. 왜 하수도가 가장 비중이 높나요? (40%)
  A. 하수도 바이러스 농도는 사람이 증상이 없어도 검출됩니다.
     즉, 무증상 감염자까지 잡아낼 수 있는 유일한 객관적 지표입니다.
     사람 행동(약 구매, 검색)과 독립적이어서 신뢰도가 높습니다.

  Q. 하나의 신호만으로 경보를 내리면 안 되나요?
  A. 안 됩니다. 구글이 2013년 검색어만 믿었다가
     실제보다 2배 과대예측해서 시스템을 폐지했습니다.
     저희는 반드시 2개 이상 신호가 동시에 임계값을 넘어야만
     경보를 발령합니다.

  Q. 실제로 보건 당국이 쓸 수 있나요?
  A. 현재는 합성 데이터 기반 검증 단계입니다.
     실제 배포를 위해서는 심평원·질병청과 MOU 및
     개인정보 처리 절차가 필요합니다. 현재는 B2G(기업→정부)
     사업화를 목표로 개발 중입니다.

  Q. 왜 AI 리포트에 Claude를 썼나요?
  A. Claude는 Anthropic이 만든 AI로, 한국어 문서 작성 능력이
     뛰어나고, 공중보건처럼 민감한 분야에서 안전한 답변을
     생성하는 것으로 검증되어 있기 때문입니다.
