▶ API (에이피아이)
  Application Programming Interface의 줄임말.

  쉬운 설명:
  식당에 비유하면 "주문서"예요.
  손님(앱)이 "서울시 오늘 경보 레벨 주세요"라고 주문서(API)를
  통해 요청하면, 주방(서버·DB)이 데이터를 꺼내서 답을 줍니다.
  API가 없으면 각 프로그램이 서로 소통할 수 없어요.

▶ Kafka (카프카)
  여러 곳에서 오는 데이터를 한 곳으로 모아주는 "메시지 전달 시스템"

  쉬운 설명:
  물류 택배 허브에 비유하면 이해하기 쉬워요.
  약국 데이터, 하수도 데이터, 검색 데이터가 각자 다른 곳에서
  만들어질 때, Kafka는 이 데이터들을 한 곳으로 모아서
  순서대로 데이터베이스에 전달해 주는 물류 허브입니다.
  (※ 현재 저희 프로젝트는 발표 안정성을 위해 Kafka 없이 DB 직접 저장)

──────────────────────────────────────────────────────

▶ TimescaleDB (타임스케일디비)
  "시간 순서"에 특화된 데이터베이스

  쉬운 설명:
  일반 데이터베이스가 "이름순 가나다 정렬 서랍장"이라면,
  TimescaleDB는 "날짜·시간순 정렬 가계부"입니다.
  "서울시 2025년 11월 3주차 감기약 구매량"처럼
  시간과 함께 저장되는 데이터를 빠르게 검색하도록 설계됐어요.
  예: 3개월치 트렌드 조회 → 일반 DB보다 10배 이상 빠름

▶ Docker (도커)
  프로그램을 "레고 블록처럼 포장해서" 어디서든 똑같이 실행되게 하는 기술

  쉬운 설명:
  "내 컴퓨터에서는 되는데 다른 컴퓨터에서 안 돼요" 문제를
  해결하는 기술입니다. 프로그램과 그 프로그램이 돌아가는
  환경(운영체제 설정, 라이브러리 등)을 하나의 박스(컨테이너)에
  포장해서 어디서든 똑같이 실행할 수 있게 해요.

▶ GKE (구글 쿠버네티스 엔진)
  구글 클라우드에서 제공하는 서버 자동 관리 시스템

  쉬운 설명:
  서버가 100개 있을 때 하나씩 관리하면 너무 힘들잖아요.
  GKE는 "이 서버가 과부하면 자동으로 서버 더 추가",
  "이 서버 죽으면 자동으로 재시작" 같은 걸 자동으로 해줘요.
  (※ 저희 프로젝트는 규모가 작아서 현재는 사용 안 함)

▶ OCR (오씨알)
  Optical Character Recognition — 이미지 속 글자를 컴퓨터가 읽는 기술

  쉬운 설명:
  사진으로 찍은 영수증을 스마트폰이 읽어서 텍스트로 바꿔주는 기능,
  그게 OCR이에요. 저희는 KOWAS PDF의 차트 그림에서
  글자 대신 "막대 색깔"을 픽셀 단위로 읽어서 숫자를 추출했어요.
  (OCR은 오타가 생기는데, 픽셀 분석은 오타가 없음)

▶ RGB 범위 마스크
  이미지에서 특정 색깔 범위에 해당하는 픽셀만 골라내는 방법

  쉬운 설명:
  사진에서 파란색만 골라내는 "색 필터" 같은 거예요.
  KOWAS 차트에서 코로나(파랑), 인플루엔자(주황), 노로(노랑)
  막대를 색깔로 구분해서 각각의 높이를 측정합니다.
  "이 픽셀이 파란색 계열이면 코로나 막대다"라는 식으로요.

▶ TFT — Temporal Fusion Transformer (티에프티)
  시간 흐름 데이터에 특화된 AI 예측 모델

  쉬운 설명:
  "시간을 이해하는 AI"예요.
  주식 차트처럼 "어제가 이렇고, 지난주가 이랬으니 다음 주는
  아마 이럴 것이다"를 예측합니다.
  특히 왜 그런 예측을 했는지 "이유"를 설명할 수 있어서
  정부 납품용으로 신뢰도가 높습니다.
  저희는 7일·14일·21일 후 독감 유행 확률을 예측합니다.

▶ Autoencoder (오토인코더)
  "평소랑 다른 이상한 상황"을 감지하는 AI 모델

  쉬운 설명:
  학교 선생님이 매일 아이들 얼굴을 보다가,
  "오늘 왠지 분위기가 달라" 하고 느끼는 감각과 비슷해요.
  Autoencoder는 평상시 데이터 패턴을 학습한 뒤,
  갑자기 그 패턴과 크게 다른 데이터가 들어오면
  "이거 이상한데?"라고 경보를 울립니다.
  예: 갑자기 검색량이 10배로 뛰면 → 이상 감지

▶ RAG (래그) — Retrieval-Augmented Generation
  AI가 답변하기 전에 관련 문서를 먼저 찾아서 참고하는 방법

  쉬운 설명:
  오픈북 시험이에요.
  일반 AI는 외운 것만 말하지만 (오답 가능),
  RAG는 답변 전에 WHO·질병관리청 공식 가이드 문서를 먼저 찾아보고
  그 내용을 근거로 답합니다.
  "이 내용은 WHO 2024 가이드라인 3페이지에서 가져왔습니다"처럼
  출처를 같이 알려줘서 신뢰도가 훨씬 높아요.

▶ LLM (엘엘엠) — Large Language Model
  텍스트를 이해하고 생성하는 초대형 AI 모델

  쉬운 설명:
  ChatGPT, Claude 같은 대화형 AI가 LLM이에요.
  엄청난 양의 글을 학습해서 사람처럼 문장을 쓰고
  질문에 답할 수 있습니다. 저희는 Claude를 써서
  경보 리포트를 한국어 문장으로 자동 작성합니다.

▶ 384차원 벡터 / 코사인 유사도
  문서 내용을 숫자로 바꿔서 "비슷한 정도"를 계산하는 방법

  쉬운 설명:
  벡터 = 문서 내용을 384개 숫자로 표현한 것.
  "독감 유행"에 관한 문서는 비슷한 숫자 배열을 가져요.
  코사인 유사도 = 두 문서가 얼마나 비슷한지 0~1 점수로 계산.
  AI가 질문을 받으면 가장 유사한 문서 3개를 골라서 참고합니다.
  (비유: 도서관에서 관련 책 찾는 과정을 자동화한 것)

▶ SSE — Server-Sent Events (서버 전송 이벤트)

▶ F1-Score (에프원 스코어)
  AI가 얼마나 잘 맞추는지 종합 점수 (0~1, 높을수록 좋음)

  쉬운 설명:
  경보를 "울려야 할 때 울렸는지"와
  "경보가 실제로 맞았는지"를 동시에 보는 점수예요.
  둘 중 하나만 잘해도 점수가 낮게 나옵니다.

  저희 결과: 0.667 (100점 만점 67점, 목표 70점에 근접)

──────────────────────────────────────────────────────

▶ Precision (프리시전) — 정밀도
  "경보를 울렸을 때 실제로 유행이 온 비율"

  쉬운 설명:
  소방차가 출동했을 때 진짜 불이 난 경우의 비율이에요.
  Precision 1.0 = 경보 울린 것 100%가 실제 유행
  → 저희 시스템이 경보 울리면 반드시 진짜라는 뜻!
  (오경보 0건)

──────────────────────────────────────────────────────

▶ Recall (리콜) — 재현율
  "실제 유행이 왔을 때 경보를 울린 비율"

  쉬운 설명:
  불이 났을 때 소방차가 출동한 비율이에요.
  Recall 0.5 = 독감이 유행한 경우 중 절반만 경보 울림
  → 절반은 잡고 절반은 놓쳤다는 뜻.
  저희는 오경보 0건을 위해 엄격한 기준을 써서 Recall이 낮음.
  (오경보 vs 놓침, 둘 다 잡기는 어려운 트레이드오프)

──────────────────────────────────────────────────────

▶ AUC-ROC (에이유씨 로씨)
  AI가 유행 상황과 평상시를 얼마나 잘 구별하는지 점수 (0~1)

  쉬운 설명:
  혈액검사로 병을 진단할 때 "정상"과 "이상"을 얼마나
  잘 구별하는지 보는 검사 정확도와 같아요.
  1.0이 완벽, 0.5는 동전 던지기 수준.
  저희 결과: 0.931 → 매우 우수한 수준

──────────────────────────────────────────────────────

▶ Walk-forward 교차검증 (워크포워드 크로스밸리데이션)
  시간 순서를 지키면서 AI를 테스트하는 방법

  쉬운 설명:
  시험 문제를 미리 보고 공부하면 안 되잖아요.
  일반 AI 테스트는 미래 데이터를 먼저 보고 과거를 예측하는
  "커닝" 문제가 생깁니다.
  Walk-forward는 반드시 과거만 학습하고 미래를 예측하게 해서
  실제와 똑같은 조건으로 테스트하는 방법이에요.
  → 숫자가 낮아 보여도 이 방법의 결과가 더 정직합니다.

▶ Gap = 4
  학습 데이터와 테스트 데이터 사이 "4주 공백"을 두는 설정

  쉬운 설명:
  1월에 공부하고 2월에 시험 치면, 공부한 게 시험에 나올 수 있어요.
  Gap=4는 "1월에 공부하고 5월에 시험 친다"처럼
  학습과 검증 사이에 4주를 비워서 미래 정보가 새어나오는 것을 막아요.

▶ Granger 인과검정 (그랜저 인과검정)
  "A가 먼저 변하면 B도 뒤따라 변하는가?"를 통계로 증명하는 검사

  쉬운 설명:
  닭이 먼저냐 달걀이 먼저냐처럼, 약국 판매가 먼저 늘어야
  독감 환자가 뒤따라 늘어야 "약국 데이터가 독감을 예측한다"고
  말할 수 있죠. Granger 검정은 이걸 통계로 증명해요.
  p < 0.05 = "우연이 아닌 것이 95% 확실하다"는 뜻.
  저희 3개 Layer 모두 p < 0.05 통과 → 과학적으로 검증됨

▶ B2G (비투지) — Business to Government
  기업이 정부·공공기관에 제품/서비스를 납품하는 사업 모델

  쉬운 설명:

  B2G는 정부한테 파는 것이에요.
  저희 목표는 질병관리청, 지자체 보건소 같은 공공기관에
  이 시스템을 납품하는 거예요.

▶ ISMS-P (아이에스엠에스-피)
  정보보호 및 개인정보보호 관리체계 인증 — 한국 정부 보안 기준

  쉬운 설명:
  건물에 소방 검사를 통과해야 영업을 할 수 있듯이,
  개인정보를 다루는 서비스는 ISMS-P 기준을 지켜야
  정부 납품이 가능해요. 보안 규정 중 제일 중요한 것이
  "SQL 인젝션 방지 (파라미터 바인딩)" — 해킹 방어 기법입니다.

──────────────────────────────────────────────────────

▶ Proxy (프록시)
  원본을 대신하는 대체 수단

  쉬운 설명:
  "약국 POS 판매 데이터"가 이상적이지만 약사회와 계약이 없어서
  대신 "네이버 쇼핑인사이트 구매 트렌드"를 대체재(proxy)로 씁니다.
  실제 약국 연결 시 collector 파일 하나만 바꾸면 즉시 전환 가능하게
  설계해뒀어요.

──────────────────────────────────────────────────────

▶ Grounding / 환각 (할루시네이션)
  AI가 사실이 아닌 것을 사실처럼 말하는 현상

  쉬운 설명:
  ChatGPT한테 "법률 정보 알려줘"라고 물으면
  가끔 존재하지 않는 판례를 지어내서 말할 때가 있어요.
  이걸 환각(Hallucination)이라고 해요.
  RAG는 이 문제를 줄이기 위해 공식 문서를 먼저 찾아보고
  그 내용만 참고해서 답을 생성합니다.
