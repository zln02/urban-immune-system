### 목적

서버에 우리 시스템을 **처음 설치해서 실행하는 과정**을 개발 경험이 없는 사람도 따라할 수 있게 정리.

### 절차

```bash
git checkout develop && git pull

```

**`docs/ops/deployment-runbook.md` 새로 작성**:

```markdown
# 배포 매뉴얼 (Deployment Runbook) v1

> 작성: 박정빈 | 2026-04-26

## 0. 사전 준비

- GCP 서버 접속 가능 (SSH 키 박정빈 보유)
- Docker 설치됨
- 프로젝트 clone 완료

## 1. 서버 접속

```bash
ssh wlsdud5035@34.64.124.90
cd ~/urban-immune-system
```

## 2. 환경 변수 설정 (최초 1회)

```bash
cp .env.example .env
nano .env
# → OPENAI_API_KEY, NAVER_CLIENT_ID, ANTHROPIC_API_KEY 등 실제 키 입력 후 저장
```

## 3. Docker 인프라 기동

```bash
sudo docker compose up -d
sudo docker compose ps
# Kafka, TimescaleDB, Qdrant 3개 healthy 확인
```

## 4. 애플리케이션 기동

```bash
# 터미널 1: 백엔드 API
source .venv/bin/activate
uvicorn backend.app.main:app --host 0.0.0.0 --port 8000

# 터미널 2: 프론트엔드 대시보드
source .venv/bin/activate
streamlit run src/app.py --server.port 8501 --server.address 0.0.0.0
```

## 5. 동작 확인

- 백엔드: http://34.64.124.90:8000/docs
- 대시보드: http://34.64.124.90:8501

## 6. 장애 대응

| 증상 | 원인 | 조치 |
|---|---|---|
| 대시보드 접속 불가 | streamlit 프로세스 죽음 | 4번 다시 실행 |
| DB 연결 에러 | Docker 컨테이너 멈춤 | `sudo docker compose up -d` |
| API 응답 없음 | uvicorn 프로세스 죽음 | 4번 다시 실행 |

## 7. 로그 위치

- 백엔드: `uvicorn` 표준 출력
- 대시보드: `streamlit` 표준 출력
- Docker: `sudo docker compose logs -f <service>`

## 8. 연락처

- 1차: 박정빈 (DevOps)
- 2차: 박진영 (PM)
```
