## 작업 1: CORS 환경변수화

### 목적
현재 `backend/app/main.py`에 하드코딩된 CORS 허용 주소를 환경변수로 이동. ISMS-P 인증 필수 조치.

### 절차

```bash
# 1) feature 브랜치 생성
git switch -c feature/ljn-cors-env

# 2) backend/app/config.py 편집 — ALLOWED_ORIGINS 필드 추가
```

**`backend/app/config.py` 편집** — 기존 Settings 클래스 안에 추가:

```python
from typing import List
from pydantic import field_validator

class Settings(BaseSettings):
    # ... 기존 필드 유지 ...

    allowed_origins: str = "http://localhost:3000,http://localhost:8501"

    @property
    def cors_origins(self) -> List[str]:
        return [o.strip() for o in self.allowed_origins.split(",") if o.strip()]
```

**`backend/app/main.py` 편집** — CORSMiddleware 부분 교체:

```python
# 기존:
# allow_origins=["http://localhost:3000", "http://localhost:8501"],
# ↓ 아래로 변경
from backend.app.config import settings

app.add_middleware(
    CORSMiddleware,
    allow_origins=settings.cors_origins,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

**`.env.example` 편집** — 파일 하단에 추가:

```
ALLOWED_ORIGINS=http://localhost:3000,http://localhost:8501
```

### 검증

```bash
uvicorn backend.app.main:app --reload --port 8000
# 다른 터미널에서:
curl -I -X OPTIONS http://localhost:8000/api/v1/signals/latest \
  -H "Origin: http://localhost:3000" \
  -H "Access-Control-Request-Method: GET"
# → 200 OK + Access-Control-Allow-Origin 헤더 확인
```

### 커밋 & PR

```bash

git commit -m "feat(backend): CORS allowlist 환경변수화 (ISMS-P 2.6 대응)"
```

## 작업 2: 감사 로그 미들웨어

### 목적
ISMS-P 인증 요건: 모든 변경성 API 요청(POST/PUT/DELETE)의 요청자·시각·응답코드 기록.

### 절차

```bash
# develop 최신화 후 새 브랜치
git checkout develop && git pull
git switch -c feature/ljn-audit-log

# 디렉토리 생성
mkdir -p backend/app/middleware
touch backend/app/middleware/__init__.py
```

**`backend/app/middleware/audit.py` 신규 작성**:

```python
"""감사 로그 미들웨어 — ISMS-P 요청자/시각/응답코드 기록."""
import logging
import time
from typing import Callable
from fastapi import Request
from starlette.middleware.base import BaseHTTPMiddleware

logger = logging.getLogger("audit")


class AuditLogMiddleware(BaseHTTPMiddleware):
    """변경성 요청(POST/PUT/PATCH/DELETE)만 감사 로그에 기록한다."""

    MUTATING_METHODS = {"POST", "PUT", "PATCH", "DELETE"}

    async def dispatch(self, request: Request, call_next: Callable):
        start = time.perf_counter()
        response = await call_next(request)
        if request.method in self.MUTATING_METHODS:
            elapsed_ms = (time.perf_counter() - start) * 1000
            client_ip = request.client.host if request.client else "-"
            requested_by = request.headers.get("X-Requested-By", "anonymous")
            logger.info(
                "AUDIT method=%s path=%s status=%d ip=%s user=%s elapsed_ms=%.1f",
                request.method,
                request.url.path,
                response.status_code,
                client_ip,
                requested_by,
                elapsed_ms,
            )
        return response
```

**`backend/app/main.py` 편집** — CORSMiddleware 아래에 추가:

```python
from backend.app.middleware.audit import AuditLogMiddleware

app.add_middleware(AuditLogMiddleware)
```

### 검증

```bash
uvicorn backend.app.main:app --reload --port 8000
# 다른 터미널에서:
curl -X POST http://localhost:8000/api/v1/alerts/generate \
  -H "X-Requested-By: test-user"
# → uvicorn 콘솔에 "AUDIT method=POST path=/api/v1/alerts/generate ..." 출력 확인
```
