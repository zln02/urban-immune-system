# ADR-001: 3-Layer 교차검증 아키텍처 채택

- **날짜**: 2026-03-09 (공모전 제안서) · 2026-04-21 (재확인)
- **담당**: @zln02 (박진영)
- **상태**: Accepted

## 맥락
감염병 조기경보는 **병원 진료 데이터**(KDCA 공식)에 의존하면 2-3주 구조적 지연이 발생한다. 대안으로 비의료 신호(검색·SNS·하수) 활용이 논의됐으나 **단일 신호는 모두 실패 전력**이 있다.

- Google Flu Trends: 검색 단일 → 2013년 2배 과대추정 후 폐기
- K-WBE (KOWAS): 하수 단일 → 바이러스 검출만, 유형·원인 해석 불가
- 소셜미디어 단독: 드라마·뉴스 노이즈

## 결정
**약국 OTC + 하수 + 검색어 3-Layer 교차검증** 을 핵심 아키텍처로 채택.

- L1 약국 OTC (네이버 쇼핑인사이트) — 구매 = 확실한 증상
- L2 하수 바이오마커 (KOWAS) — 사람 행동과 독립, 무증상 감지
- L3 검색 트렌드 (네이버 DataLab) — 실시간성
- 앙상블 가중치: W1=0.35, W2=0.40, W3=0.25
- 경보 룰: **2개 이상 Layer** 임계값 초과 시 YELLOW 이상 발령

## 대안
| 대안 | 기각 이유 |
|---|---|
| 단일 Layer (검색만) | GFT 실패 사례 재현 위험 |
| 2-Layer (하수+검색) | Deng et al. 2026 이미 발표, 차별성 없음 |
| 4-Layer (+SNS) | SNS 크롤링 법무 이슈, 노이즈 >> 신호 |
| 의료 데이터 포함 | 개인정보 영향, ISMS-P 부담 |

## 결과 (지금 알 수 있는 장단점)

### 장점
- 각 Layer 의 약점을 상호 보완
- 하수(L2) 가 앵커 역할 → FP 감소
- 한국 3대 공공/민간 데이터 소스 활용 → 해외 대비 차별성
- 설명 가능성 (TFT Attention 으로 Layer 기여도 표시)

### 단점
- 데이터 소스 3개 유지 부담 (API 키·법무·수집 인프라)
- 네이버 API 상업 재판매 라이선스 미해결 (비즈모델 리스크)
- 단일 시즌 합성 데이터만으로는 과적합 의심 가능
