# FAQ-001: 왜 3-Layer 교차검증인가? (난도: E)

## 핵심 답변 (30초)

> 감염병 신호는 단일 데이터 소스로 판단하면 오경보가 난무합니다. Google Flu Trends(2008-2015) 가 검색어만으로 독감을 예측했다가 2013년 실제 대비 2배 과대추정으로 폐기된 사례가 대표적입니다. 저희는 (1) 약국 OTC 판매 — 구매는 확실한 증상 존재, (2) 하수 바이오마커 — 무증상자도 감지, (3) 검색 트렌드 — 실시간성 최고. 이 세 신호가 **동시에** 올라가야 경보를 내도록 설계해 오경보를 필터링합니다. 특히 하수 데이터는 사람의 검색·구매 행동과 독립적이므로 교차검증의 앵커 역할을 합니다.

## 상세 (면접 깊게)

### 왜 3개인가 (2개가 아닌)
- Deng et al. (2026, *Engineering*): 하수+검색 2-Layer 퓨전 논문 존재
- 저희는 여기에 **약국 OTC** 추가 → 소비자 행동 데이터 (구매 = 증상 존재 확정)
- 결과: 단일 2-Layer 가 놓치는 무증상·경증 케이스 커버

### 각 Layer 의 역할
| Layer | 데이터 | 선행 시간 | 약점 |
|---|---|---|---|
| L1 약국 OTC | 네이버 쇼핑인사이트 | ~2주 | 자가격리 시 방문 감소 |
| L2 하수 | KOWAS | ~3주 | 바이러스 종류 제한 |
| L3 검색 | 네이버 DataLab | ~1주 | 미디어 노이즈 |

→ 각자 약점이 상호 보완됨 (하수는 격리와 무관, 검색은 자가격리자도 유지)

### 경보 발령 규칙
**2개 이상 Layer 가 임계값 초과** 해야 YELLOW 이상. 단일 Layer 급등은 무시. → FP 감소.

## 심사위원 재질문 예상

**Q. "네이버 API 상업 이용 약관 재판매 가능한가요?"**
→ "현 단계에선 연구·데모 목적 한정. 상용화 전 네이버 개발자센터 공식 문의·협업 제안 예정."

**Q. "Deng 2-Layer 가 이미 있는데 우리 차별점은?"**
→ "(1) 한국 로컬라이제이션 (KOWAS + 네이버), (2) 약국 OTC 추가 → 소비자 행동 데이터, (3) RAG-LLM 경보 리포트."
