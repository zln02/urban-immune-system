## 카테고리 1 — 검증·재현성 (★ 가장 자주 나옴)

### Q1. F1 0.882 어떻게 그렇게 높게 나왔어요? 합성 데이터 아닌가요?

**A**: 17개 시·도 walk-forward 백테스트, gap 4주, 5-fold. 실측 입니다. 시간 순으로 학습·검증 분할해서 미래 누출 막았습니다. `python -m ml.reproduce_validation` 1줄 명령으로 누가 돌려도 같은 수치 나옵니다. F1·Recall·Lead 모두 직전 baseline 대비 향상(F1 0.841→0.882, Recall 0.768→0.837, Lead 5.9→6.47주).
- 슬라이드: **#/20 (S11)** + **#/12 (S07C)**
- 백업: 합성 학습 결과는 F1 0.667 (S07 보여드린 정직 정정 수치). 실측 0.882은 현실에 가까움.

### Q2. Recall 0.837은 목표 0.85에 약간 미달인데 놓침이 너무 많지 않나요?

**A**: 게이트 B로 단일신호 차단하니 일부 진짜 위험을 GREEN으로 떨어뜨립니다. 그래서 Recall이 목표 0.85에 약간 못 미친 0.837. 다만 직전 baseline 0.768 대비 +0.069 향상. 대신 Precision 0.949 — 경보 신뢰도 95%, FAR 0.206 (게이트 OFF는 0.602로 65.8% 감소). 보건당국이 가짜 경보로 신뢰 잃는 게 더 큰 손실이라고 판단했습니다. 게이트 임계 완화 시 Recall 0.90 이상 달성 가능하나 FAR 1.5배 trade-off.
- 슬라이드: **#/20 (S11)** + **#/11 (S07B)**

### Q3. Granger p=0.021은 무슨 의미인가요?

**A**: 통계적 선행성 검증입니다. p가 0.05 미만이면 "이 신호가 임상 확진보다 진짜로 먼저 움직인다"는 가설이 5% 미만 오류로 통계적으로 뒷받침된다는 뜻입니다. 우리는 0.021 — 신뢰 구간 안에 들어옵니다. L3 검색은 0.007로 더 강합니다.
- 슬라이드: **#/20 (S11)**

### Q4. 6.5주 선행이 임상적으로 의미 있나요?

**A**: 임상 확진 피크 약 6.5주 (정확히 6.47주, 17지역 평균) 전 YELLOW 발령이면 역학조사관 사전 배치, 응급실 트리아지 강화, 마스크 권고 등 사전 대응 시간을 한 달 반 확보합니다. 가장 빠른 곳은 세종 9주, 부산·제주 8주, 서울 7주, 가장 짧은 12개 지역도 6주. 의료적 진단을 대체하는 게 아니라 정책 결정 보조 자료입니다.
- 슬라이드: **#/20 (S11)** + **#/2 (S02)**
- 백업: KDCA 자문 회신 후 임상 활용 적합성 정식 검증 예정 (S13B 자문 트리거).

## 카테고리 2 — 모델·코드 디테일

### Q5. XGBoost 말고 TFT는 왜 안 쓰셨어요? 현재 production 모델이 뭐예요?

**A**: 현재 prod는 XGBoost 회귀 + 게이트 B입니다. F1 0.882·Precision 0.949·MCC 0.595 안정적. TFT는 PoC 단계 — 실데이터 17지역 학습 완료해서 70% (S09). 148K 파라미터, val_loss 5.48. attention 가중치를 출력해 **어느 신호가 결정에 기여했는지** 보여주는 게 핵심. 보건당국 신뢰 확보용입니다. 데이터 추가 누적 후 prod 전환 예정.
- 슬라이드: **#/15 (S07F)** + **#/17 (S09)**

### Q6. attention top-3가 기온·OTC·하수네요? 검색은 왜 4위인가요?

**A**: 좋은 질문입니다. 합성 학습에서는 검색이 1위였는데 실데이터에서는 4위로 밀렸습니다. 검색 단독으로 흔들리는 노이즈가 많기 때문입니다. **이게 게이트 B의 정당성을 직접 입증**합니다 — 검색 단독으로 경보 내면 실패. 기온은 메타 시그널 (시즌성), OTC·하수가 실제 의료 신호 1·2위. 정확히 우리 설계 의도 반영.
- 슬라이드: **#/15 (S07F)** + **#/11 (S07B)**

### Q7. Autoencoder는 어디서 쓰이나요?

**A**: 신종 패턴 감지 — 라벨 없이 학습합니다. 정상 데이터를 압축·복원하는데 처음 보는 패턴은 복원 실패 → 그 오차로 이상 감지. 99% 분위수 이상만 알람 (`ff17dfa` 핫픽스로 95p→99p 상향, 17지역 inference 16/17 false positive→1/17 정상화). **다음 팬데믹** 같은 신종 시그널 잡는 데 핵심. 인플루엔자 학습된 모델로 코로나 잡을 수 있는 이유입니다.
- 슬라이드: **#/15 (S07F)** + **#/24 (S14)**

### Q8. walk-forward 검증이랑 일반 K-Fold랑 뭐가 달라요?

**A**: 일반 K-Fold는 데이터를 무작위 섞어서 분할합니다. 시계열에서는 미래 데이터가 과거 학습에 들어갈 수 있어 발표용 점수만 잘 나옵니다. walk-forward는 시간 순서대로 분할하고 4주 갭으로 한 달치 미래를 강제 차단합니다. 그래서 진짜 실전 성능을 측정합니다.
- 슬라이드: **#/12 (S07C)**

## 카테고리 3 — 데이터·확장성

### Q9. KOWAS 하수 데이터를 PDF에서 픽셀로 추출한다고요? 정확도가 어떻게?

**A**: KOWAS가 API를 비공개해서 PDF 그래프의 막대 색상 RGB를 직접 측정해 농도를 추출합니다. `PATHOGEN_COLOR_RANGES` 코드 한 줄로 covid·influenza·norovirus 색상 범위 정의. 72개 PDF 자동 다운로드되어 있고, KCDC 표본감시 결과와 비교 검증 시 R² 0.85 이상 일치. 다만 정식 활용 시 KOWAS 원본 데이터 액세스 필요해 KDCA 자문 요청 항목입니다.
- 슬라이드: **#/9 (S07A)** + **#/23 (S13B)**

### Q10. 다른 질병으로 확장 가능한가요? 코로나는 어떻게?

**A**: 코드 한 줄 추가로 가능합니다. `PATHOGEN_COLOR_RANGES`에 covid 색상 범위 추가, `SIDO_ALL` 지역 리스트 교체, 가중치 `w4` 한 칸 더하면 새 신호층 추가. 이 모델은 인플루엔자 전용이 아니라 다음 팬데믹용입니다. Autoencoder가 핵심 — 학습 안 한 패턴도 잡습니다.
- 슬라이드: **#/6 (S05A)** + **#/24 (S14)**

### Q11. 네이버 API 무료 한도 넘으면 어떻게 하나요?

**A**: 네이버 쇼핑인사이트와 데이터랩은 일 25,000회 무료. 17지역 × 키워드 5개 × 매주 1회 = 85회/주. 한도 안에 충분히 들어옵니다. 한도 늘어나면 **유료 전환 + 가격 모델 Tier C PoC 1건 (1,000~3,000만)** 비용에 포함시켜 우리가 부담하는 구조 설계.
- 슬라이드: **#/25 (S15)**

## 카테고리 4 — 법규·보안

### Q12. 개인정보보호법 어떻게 처리하셨어요?

**A**: 모든 입력이 출처 단계에서 **시·도 단위 집계 지수** (수십만~수천만 인구 평균값) 만 수신됩니다. 개인 식별자 직접 처리 없고, 다른 정보와 결합해도 특정 개인 식별 불가. 자체 검토로 개인정보보호법 제2조 1호 비해당 판정. KISA 사전 컨설팅 자문 발송해 정식 의견 받을 예정. DPIA 초안도 작성됐습니다 (자문 패키지 22번 파일).
- 슬라이드: **#/23 (S13B)**

### Q13. ISMS-P 인증 받으셨나요?

**A**: 학부생 PoC라 정식 인증은 Phase 3 (2027 H1) 예정입니다. 현재 25% 준비 — K8s `runAsNonRoot`, 권한 상승 차단, 읽기 전용 루트 파일시스템, Linux capability 모두 drop, Kafka acks=all + retries=3, .env 격리 + Pydantic 검증, 면책 조항 강제 삽입(2.9 / EU AI Act 13·14). KISA 사전 컨설팅으로 통제 우선순위 자문 받습니다.
- 슬라이드: **#/26 (S16) Q&A 부록**

## 카테고리 5 — B2G 사업·가격

### Q14. 정부에 어떻게 팔겠다는 거죠? 학부생 프로젝트가?

**A**: 캡스톤 단계라 직접 판매는 아닙니다. 3-tier 가격 모델 — Tier A 구독 (지자체 연 2~6천만, 광역 1~3억), Tier C **PoC 1건 1,000~3,000만**, Tier B 건당 50~300만. 6개월 안 목표는 PoC 1건 수주. KDCA·서울시·경기도 단계적 접근. 발표 끝나면 외부 자문 자료 발송, 6월 ILINet PoC 협력 의향서 받는 게 다음 단계입니다.
- 슬라이드: **#/25 (S15)** + **#/23 (S13B)**

### Q15. BlueDot, CDC NWSS 같은 글로벌 시스템과 차별점이 뭐예요?

**A**: BlueDot은 뉴스 기반, CDC NWSS는 하수만, KAIST와 Xu 2025는 두 신호. 모두 단일 또는 이중 신호입니다. 우리는 **3계층 교차검증 + 게이트 B 코드 강제**가 차별점. KIPRIS 검색에서 OTC+하수+검색 3계층 동시 결합 특허 미확인. 다음 단계는 변리사 자문 후 출원 검토입니다.
- 슬라이드: **#/21 (S12)** + **#/22 (S12B)**

## 카테고리 6 — 협업·완성도

### Q16. 5명이 어떻게 역할 나눴어요?

**A**: PM/ML 박진영, 백엔드 이경준, 데이터 이우형, 프론트 김나영, DevOps/QA 박정빈. **Claude Code 에이전트로 병렬 협업**했습니다 — 각자 모듈 디렉토리에서 `claude` 세션 열면 해당 모듈 CLAUDE.md 자동 로드되어 역할 특화 에이전트로 작동. 5명이 동시에 5개 모듈 진행 가능했던 비결입니다.
- 슬라이드: **#/26 (S16)**

### Q17. 완성도가 70%, 80% 표기인 이유는요? 100%라고 안 하는 이유?

**A**: 정직성이 본질이라고 판단했습니다. TFT 70%는 실데이터 학습은 끝났지만 KCDC 실측 confirmed_cases를 타깃으로 하는 fine-tuning 30%가 남았다는 뜻. RAG 90%는 임베딩 17건 완료지만 가이드 더 늘리는 작업 10% 남음. **100%라고 하면 교수님이 깊이 질문하셨을 때 답변 막힙니다**. 우리가 어디까지 했는지 정확히 아는 게 신뢰입니다.
- 슬라이드: **#/17 (S09)**
