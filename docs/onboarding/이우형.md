## ✅ 완료 증명 (5단계 E2E 테스트)

```bash
# 1️⃣ Consumer 백그라운드 실행
python -m pipeline.collectors.kafka_consumer > /tmp/consumer.log 2>&1 &
CONSUMER_PID=$!
sleep 2

# 2️⃣ Producer 로 테스트 메시지 1건 발행
python -c "
from pipeline.collectors.kafka_producer import send_signal, TOPIC_L1
send_signal(TOPIC_L1, '서울특별시', 'L1', 42.5, raw_value=180, source='manual_test')
print('✅ Producer 메시지 발행 완료')
"

# 3️⃣ 3초 기다렸다가 DB row 증가 확인
sleep 3
sudo docker exec uis-timescaledb psql -U uis_user -d urban_immune -c "SELECT count(*) FROM layer_signals;"
# → count 가 0 → 1 로 증가했으면 성공 🎉

# 4️⃣ 실제 저장된 row 내용 확인
sudo docker exec uis-timescaledb psql -U uis_user -d urban_immune -c "SELECT time, layer, region, value, source FROM layer_signals ORDER BY time DESC LIMIT 5;"

# 5️⃣ Consumer 종료
kill $CONSUMER_PID

# 6️⃣ 테스트·린트·커밋·PR
pytest tests/ -q
ruff check pipeline/
git add -A
git commit -m "feat(pipeline): Kafka Consumer → TimescaleDB 적재 (closes #4)"
git push -u origin feature/lwh-kafka-consumer
gh pr create --base develop --fill
```

## 🆘 자주 막히는 지점

| 증상 | 해결 |
|---|---|
| `kafka.errors.NoBrokersAvailable` | `KAFKA_BOOTSTRAP_SERVERS=localhost:9092` 확인 |
| 토픽 없다고 에러 | Producer 가 없는 토픽 생성 → Consumer 먼저 실행 말고 Producer 먼저 |
| DB 에 row 안 쌓임 | `/tmp/consumer.log` 에 INSERT 에러 있는지 확인 |
| `asyncpg.exceptions.InvalidPasswordError` | `.env` 의 `DB_PASSWORD` 확인 |
| Consumer 가 같은 메시지 반복 소비 | offset commit 실패 — `enable_auto_commit=False` + 수동 commit 확인 |
