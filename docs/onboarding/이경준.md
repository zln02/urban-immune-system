## 🐛 발견한 버그 (이게 당신이 고칠 것)

### 1. 테이블명 불일치
```python
# backend/app/api/signals.py 현재
FROM signals          # ← 존재하지 않음
```
```sql
-- infra/db/init.sql 실제
CREATE TABLE layer_signals (...)   -- ← 진짜 테이블
```

### 2. 컬럼명 불일치
| 코드가 쓰는 컬럼 | 실제 DB 컬럼 |
|---|---|
| `recorded_at` | `time` |
| `layer` (otc/wastewater/search) | `layer` (L1/L2/L3/AUX) |

### 3. pattern 정규식 불일치
```python
# 현재: pattern="^(otc|wastewater|search)$"
# 실제 DB 값: L1 / L2 / L3 / AUX
```

**첫 프롬프트** (그대로 복사):
```
backend/app/api/signals.py 를 수정해줘.
infra/db/init.sql 의 layer_signals 테이블 스키마에 맞춰서:

1. FROM signals → FROM layer_signals 로 테이블명 변경
2. recorded_at → time 컬럼명 변경
3. layer pattern 을 ^(L1|L2|L3|AUX)$ 로 수정
4. Pydantic 응답 모델 LatestSignalsResponse, TimeseriesResponse 정의
5. SQL 파라미터 바인딩 유지 (:layer, :region, :days)
6. 변경 전에 뭘 바꿀지 먼저 설명해줘.

infra/db/init.sql 의 layer_signals 스키마:
- time TIMESTAMPTZ
- layer VARCHAR(10)  -- 'L1' | 'L2' | 'L3' | 'AUX'
- region VARCHAR(50)
- value DOUBLE PRECISION
- raw_value DOUBLE PRECISION
- source VARCHAR(100)
```

Claude 가 설명 주면 **"좋아 수정해"** 라고 답.

## 🆘 자주 막히는 지점

| 증상 | 원인 | 해결 |
|---|---|---|
| `ModuleNotFoundError` | `.venv` 비활성 | `source .venv/bin/activate` |
| `Connection refused` 5432 | Docker DB 꺼짐 | `sudo docker compose up -d` |
| `relation "signals" does not exist` | 버그 맞음 — 수정 대상 | Claude 에게 위 프롬프트 |
| `pytest fail` | 기존 테스트 확인 | `pytest tests/ -v` 로 어느 테스트 실패인지 확인 |
| `git push permission denied` | GitHub 인증 | `gh auth login --web && gh auth setup-git` |
