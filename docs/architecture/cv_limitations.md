# CV Fold NaN — 26주 단일시즌 한계와 6주 보완안

## 요약

`synthetic_hardened` 데이터셋(104주, 양성 15개, 양성율 14.4%)에 TimeSeriesSplit(n_splits=5, gap=4)을 적용한 결과, 5개 fold 중 **4개(Fold 1·2·3·5)가 F1/Precision/Recall NaN**을 반환했다.
유일하게 유효한 Fold 4의 F1=0.667(Precision=1.0, Recall=0.5)이 cv_mean으로 집계됐다.

이는 모델 결함이 아닌 **단일 독감 시즌에 양성 사례가 집중되는 데이터 구조적 한계**에서 비롯된 것이다.
실 신뢰 근거는 합성 CV가 아닌 **17지역 walk-forward 실측 백테스트 F1=0.907(Precision=0.940, Recall=0.882)**이다.

## 데이터 제약 및 CV 설정

| 항목 | 값 |
|------|-----|
| 데이터셋 | `synthetic_hardened` |
| 전체 주수 | 104주 (week 0 ~ 103) |
| 양성 레이블 수 | 15개 (14.4%) |
| alert_threshold | 확진자 ≥ 70명 (t+2주 선행) |
| 특성 컬럼 | l1_otc, l2_wastewater, l3_search, temperature, humidity |
| CV 방식 | TimeSeriesSplit(n_splits=5, gap=4) |
| gap 의미 | val 직전 4주를 훈련에서 제외 — 미래 누출 방지 |

양성 15개는 독감 성수기(시즌 피크)에 집중 분포한다. 5-fold 분할 시 대부분의 fold는 훈련 또는 검증 구간이 비성수기에 해당해 양성 레이블이 0개가 된다.

- NaN 판정 코드: `ml/xgboost/model.py:230` — `zero_division=np.nan`
- CV 설정 코드: `ml/xgboost/model.py:210`

> Fold 2·3·5 비고: val 구간이 비성수기 → 레이블 단일 클래스(전부 음성) → F1 정의 불가.

## Fold 4만 유효한 이유 (정량 근거)

Fold 4의 val 구간(week 70~86)이 독감 성수기에 해당한다.

- **훈련 구간(week 0~69)**: 양성 7개 포함 → 모델이 양성 패턴 학습 가능
- **검증 구간(week 70~86)**: 양성 8개 포함 → F1 계산 가능

결과:
- F1 = 0.667 (Precision=1.0, Recall=0.5)
- AUC-ROC = 0.931 — 클래스 분리 능력 우수
- Precision=1.0: 모든 양성 예측이 정확 (FP=0)
- Recall=0.5: 양성 8개 중 4개만 탐지 (FN=4) — 임계값 조정으로 개선 가능

Recall 0.5의 원인: 훈련에서 본 양성 수(7개)가 절대적으로 적어 결정 경계가 보수적으로 형성됨.

## Q&A

**Q: CV 결과가 하나의 fold만 유효한데 신뢰할 수 있나?**

A: CV NaN은 모델 성능 문제가 아니라 단일 독감 시즌(104주)에 양성 15개가 집중 분포하는 데이터 구조 문제다. 5-fold 시 대부분 구간이 비성수기에 배정되어 양성 레이블이 0개가 된다. 실제 모델 신뢰 근거는 합성 CV가 아닌 **17지역 walk-forward 실측 백테스트(F1=0.907)**이며, 이는 실제 2025-09~2026-03 독감 시즌 데이터 기반이다. 두 검증 결과를 분리해서 해석해야 한다.

**Q: Fold 4 Recall=0.5가 낮지 않나?**

A: 훈련 구간 양성 7개라는 극히 제한된 샘플에서 학습한 결과로, 결정 경계가 보수적으로 형성됐다. 같은 이유로 Precision=1.0(FP=0)을 달성했다. 실측 17지역 환경에서는 누적 데이터로 Recall=0.882까지 향상됐다. 임계값을 0.5→0.3으로 조정하면 합성 환경에서도 Recall 개선 여지가 있다.

**Q: NaN이 발표 지표에 영향을 주지 않나?**

A: 슬라이드 및 공식 발표 지표는 **17지역 walk-forward 실측 백테스트 수치(F1=0.907)**를 사용한다. `ml/outputs/validation.json`의 `cv_mean_f1=0.667`은 NaN을 제외한 유효 fold 평균이며, 합성 검증 컨텍스트에서만 참조한다. 혼용하지 않도록 validation.json 내 stages 구조로 분리돼 있다.

**Q: TimeSeriesSplit gap=4 설정 근거는?**

A: 독감 경보의 목표 리드타임이 2~3주이므로, gap 4주는 val 직전 4주를 훈련 제외함으로써 미래 신호 누출을 방지한다. 시계열 예측에서 gap 없는 CV는 look-ahead bias를 유발해 성능을 과대평가한다. 이 gap이 유효 양성 수를 더 줄이는 부작용이 있으나, 보수적 검증 원칙을 우선한다.
