## Page 1 — System Overview

### Title
Urban Immune System (UIS) — 비의료 다중 신호 교차검증 감염병 조기경보 AI

### Abstract (English)
The Urban Immune System (UIS) is a B2G-oriented AI system that fuses three non-medical signals — pharmacy OTC purchases (Naver Shopping Insight), wastewater biomarker concentrations (KOWAS), and search trends (Naver DataLab) — to deliver region-level infectious disease early warnings 1–3 weeks before clinical confirmation. The system enforces a 2-of-3 cross-validation gate (Gate B) that reduces false-alarm rate by 58.5% versus single-layer alerts. Region-tiered threshold calibration adapts to weak-signal regions (Chungbuk, Daegu, Gyeongbuk) without modifying the algorithmic claim. Validated on 17 administrative regions with 26-week walk-forward backtesting: F1=0.907, Recall=0.882, FAR=0.250 (Gate ON), lead time 6.47 weeks.

### 요약 (한국어)
Urban Immune System(UIS)은 약국 OTC 구매(네이버 쇼핑인사이트)·하수 바이러스 농도(KOWAS)·검색 트렌드(네이버 데이터랩) 3계층 비의료 신호를 융합해 임상 확진보다 1–3주 선행하는 감염병 조기경보를 17개 시·도 단위로 발령하는 B2G 지향 AI 시스템이다. 2-of-3 교차검증 게이트(Gate B)로 단일 신호 대비 오경보율을 58.5% 감소시켰으며, 약신호 지역(충북·대구·경북)에 대한 지역별 임계값 차등 보정으로 알고리즘 청구범위는 보존한 채 recall을 향상시켰다. 17개 시·도 26주 walk-forward 백테스트 검증 결과: F1=0.907, Recall=0.882, FAR=0.250(Gate ON), Lead time 6.47주.

### Key Differentiators

| # | Feature | Innovation |
|---|---------|-----------|
| 1 | 3-layer non-medical signal integration | OTC(L1) + Wastewater(L2) + Search(L3) — 임상 확진 없이 조기 신호 포착 |
| 2 | Gate B cross-validation rule | 2개 이상 계층 동시 임계 초과 시에만 경보 → FAR −58.5% |
| 3 | Region-tiered calibration | 17개 시·도 약신호 지역 composite/layer threshold 차등 (calibration only) |
| 4 | RAG natural language reporting | GPT-4o + Qdrant 벡터 검색, XAI 메타데이터 강제 기록 |

### Statistical Rigor

- **Walk-forward CV**: 26-week single-season, gap=4w buffer, 5 folds × 17 regions = 85 evaluations
- **Granger causality** (composite signal vs clinical confirmation): p=0.021 (significant)
- **Gate B trade-off**: Recall_no_gate=0.904 → Recall_gate=0.882 (−2.4%) / FAR_no_gate=0.602 → FAR_gate=0.250 (−58.5%)
- **Region distribution**: 충북 recall 0.529→0.941 (+0.412, composite-level regional threshold)

## Page 3 — Known Limitations (정직)

### Data Freshness Constraints

| Source | Layer | Expected lag | Cadence |
|--------|-------|--------------|---------|
| Naver Shopping Insight | L1 OTC | T-2 days | Daily publish |
| Naver DataLab | L3 Search | T-2 days | Daily publish |
| KOWAS (covid/influenza/norovirus) | L2 Sewage | T-7~10 days | Weekly Tue PDF |
| KMA Temperature | AUX | T-0 | Hourly |

운영 의미: lag이 위 기준 초과 시 collector 장애. 이내면 정상.

### Generalization Limits

- **단일 시즌 (26주)**: 2025–2026 인플루엔자 1개 시즌. Phase 3에서 2시즌 이상 누적 후 외부 검증 예정.
- **17개 시·도 한정**: 시·군·구 단위 미지원 (Phase 3 HIRA OpenAPI 도입 시 분리).

### Signal Quality Caveats

- **L1/L3 전국 단일값 broadcast**: 네이버 API region 미지원으로 전국 집계 → 17개 시·도 fan-out. HIRA 도입 후 지역 분리 (Phase 3).
- **L2 KOWAS PDF 수동 추출**: Selenium 자동 크롤링은 Phase 3 우선 과제. 현 PDF 수동 다운로드.
- **TFT-real val_loss=5.48 (epoch 5)**: 26주 한계로 발산 — 발표 데모는 XGBoost(주모델)만 사용. TFT는 PoC 위치, 데이터 누적 후 prod 전환 예정. 자세한 분석은 docs/architecture/cv_limitations.md 참조.

### Other

- **RAG 정량 평가 미완**: BLEU/ROUGE 기반 generation quality 평가 향후 과제.
- **충북 L2 신호 약함**: AUPRC 0.917~0.980로 모델 자체는 양호하나 composite<30 구간에서 약신호 발생 → composite-level regional threshold로 부분 보정.
