# 가격 모델 (Pricing Model) — Urban Immune System

> **상태**: Draft (2026-04-28, 중간발표 D-2 기준 stub)
> **소유자**: 박진영 (PM/ML Lead)

## 1. 모델 개요

3-tier 모델로 고객군의 예산·구매 성향에 맞춰 분리한다.

| Tier | 형태 | 가격 범위(연) | 타깃 |
|---|---|---|---|
| **A. 구독 (Subscription)** | SaaS — 대시보드 + API + 경보 | 2,000만 ~ 3억 | 지자체·광역·질병관리청 |
| **B. 건당 (Per-Report)** | RAG 리포트 PDF + 데이터 패키지 | 50만 ~ 300만/건 | 민간 보험·연구기관·언론 |
| **C. PoC (1회성)** | 30~90일 시범 운영 + 분석 보고서 | 1,000만 ~ 3,000만 | 신규 기관 첫 도입 |

## 2. Tier A — 구독

### 2.1 산정 근거

| 변수 | 설명 | 기여도 |
|---|---|---|
| **인구 규모** | 인구 1만 명당 월 ~30만원 (산정 기준선) | 50% |
| **데이터 통합 깊이** | API 연동 수(L1~L4·기상·내부 EMR) | 25% |
| **응답 SLA** | p95 < 500ms · 99.9% 가동시간 옵션 | 15% |
| **AI 리포트 한도** | 월 RAG 리포트 생성 횟수 (50/200/무제한) | 10% |

### 2.2 구간

| 등급 | 대상 | 연 가격 | 포함 |
|---|---|---|---|
| Basic | 기초 지자체 (50만 미만) | **2,000~3,000만** | L1+L3, 월 50 RAG |
| Standard | 기초 지자체 (50만 이상) · 광역 산하 | **3,000~6,000만** | L1+L2+L3, 월 200 RAG, SLA 99.5% |
| Premium | 광역 본청 · 질병관리청 | **1억~3억** | 전 계층 + 자체 모델 튜닝 + SLA 99.9% + 24/7 지원 |

### 2.3 상한·하한
- 하한: **연 2,000만** (운영 인건비 분담선)
- 상한: **연 3억** (단일 기관, 광역·중앙)

## 3. Tier B — 건당 리포트

| 패키지 | 내용 | 가격/건 |
|---|---|---|
| **Snapshot** | 단일 지역·1주 RAG 리포트 PDF (4쪽) | 50만 |
| **Region Pack** | 17개 광역 비교 + 4주 추세 + Granger 인과 | 150만 |
| **Custom Risk** | 보험사 약관·연구 가설별 맞춤 분석 | 200~300만 |

> 💡 **활용 사례**: 손해보험사가 독감 시즌 청구 급증 위험을 분기별로 평가, 언론사가 광역별 위험 비교 기사 작성.

## 4. Tier C — PoC

### 4.1 표준 PoC 패키지

| 항목 | 내용 |
|---|---|
| **기간** | 30 / 60 / 90일 (선택) |
| **포함** | 단일 광역 1곳 실시간 모니터링 + 경보 자동발송 + 종료 보고서 |
| **인력** | UIS 팀 PM 1 + ML 1 + DevOps 0.5 (파트타임) |
| **가격** | **30일 1,000만 / 60일 2,000만 / 90일 3,000만** |
| **전환 인센티브** | PoC 종료 후 12개월 내 Tier A 계약 시 PoC 비용의 50% 차감 |

### 4.2 종료 산출물
1. F1·MCC·AUPRC walk-forward 검증표 (해당 광역 실데이터)
2. 4쪽 RAG 경보 리포트 샘플 5건
3. ISMS-P 적용성 의견서
4. Phase 2 운영 인수인계서

## 5. 가격 정책 원칙

- **공공기관 절대 무료 사용 거절** — B2G 납품의 신뢰성·지속성을 위해 PoC 단계부터 유상.
- **할인은 다년 계약**으로만 — 3년 계약 시 15%, 5년 계약 시 25%.
- **국정감사·감염병 위기 단계 격상 시 가격 동결** — 사회적 책임 조항.
- **민감 데이터 처리** 옵션은 별도 ISMS-P 부속 계약 필요.
