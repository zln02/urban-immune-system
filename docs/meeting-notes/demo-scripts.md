## ❓ 예상 Q&A (교수님·심사위원)

### Q1. "F1 0.71 에서 0.643 으로 낮아진 건 퇴보 아닌가?"
> **A**: 퇴보가 아니라 **더 엄격한 평가** 입니다. 공모전 수치는 단일 임계값 최적화 후 가장 좋은 값만, 현재는 walk-forward CV + MCC·AUPRC 병기. B2G 납품 시 재현 가능한 숫자가 본질입니다.

### Q2. "지금 실시간으로 질병청 데이터 쓰나?"
> **A**: 아닙니다. 현재는 합성 데이터(seed=42)와 KDCA ILINet CSV 수동 다운로드. 실시간 수집은 Phase 2 기말 목표.

### Q3. "Precision 1.00 없어진 이유는?"
> **A**: 클래스 불균형(양성 5%)에서 threshold 를 높이면 자동으로 Precision=1.00 가능. MCC·AUPRC 가 불균형 데이터 표준 지표입니다.

### Q4. "TFT 말고 TimesFM 같은 최신 모델 왜 안 써?"
> **A**: Foundation Model 들은 공중보건 도메인 검증 논문이 적습니다. TFT 는 말라리아·뎅기·독감 검증 다수. B2G 는 "왜 경보?" 답변 가능성이 정확도만큼 중요.

### Q5. "팀원 4명은 뭘 했나? 박진영 원맨쇼 아닌가?"
> **A**: 팀원 4명 각자 이슈 #3-6 배정. CODEOWNERS 모듈별 승인. W18 에 각자 PR 1건 목표. 발표에서도 박정빈이 법·규제, 김나영이 Demo 공동 진행.

### Q6. "네이버 API 상업 이용 되나?"
> **A**: W18 에 네이버 개발자센터 공식 문의 접수. 상업 시 파생 지표(z-score 등)만 제공하는 방식 검토 중.

### Q7. "K8s GKE 왜 포기했나?"
> **A**: 6주 안에 WIF·Ingress·TLS 세팅이 불안정. Phase 3 여름에 착수. 현재는 Docker Compose 로 검증.

### Q8. "Google Flu Trends 실패와 어떻게 다른가?"
> **A**: GFT 는 **검색 단일 신호**. 저희는 하수 Layer 가 사람 행동과 독립이라 False Alarm 필터. 3 Layer 전부 p<0.05 유의.

### Q9. "상업화 가능성은 몇 %?"
> **A**: 6개월 내 현실 시나리오는 35~45%. 조건: 법인 설립 + KOWAS MOU + 네이버 상업 이용 승인. 보수적 예상: 정부과제 2~5천만 + PoC MOU 1건.

### Q10. "중간발표 이후 제일 먼저 할 일은?"
> **A**: KDCA ILINet 실데이터 수집 → LightGBM 실측 재측정. 현재 합성 기반 0.643 이 실데이터에서 어떻게 나오는지가 Phase 2 핵심.
