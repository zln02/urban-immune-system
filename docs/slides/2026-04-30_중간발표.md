## Slide 3 — 3-Layer 아이디어

| Layer | 데이터 | 선행 시간 |
|---|---|---|
| 💊 약국 OTC | 네이버 쇼핑인사이트 | ~2주 |

**3개 신호가 동시에 오르면 진짜 경보**. 단일 신호는 노이즈, 교차 검증으로 오경보 최소화.

### 왜 3개인가?
Google Flu Trends (2008-2015) 는 검색 단일 신호 → 2013년 **실제 대비 2배 과대추정** 후 폐기.
저희는 하수(L2) 가 **사람 행동과 독립**인 앵커 역할을 해서 False Alarm 필터.

## Slide 6 — 핵심 기술 ①: TFT (시계열 예측)

**Temporal Fusion Transformer** (Google, 2021)
- 여러 시계열 + 정적 피처 + 미래 공지 이벤트 종합
- Attention 메커니즘으로 **어느 신호가 예측에 기여**했는지 해석 가능

### 왜 TFT 인가? (TimesFM·Moirai·Chronos 2024 등장 이후에도)
- Foundation model 들은 **공중보건 도메인 검증 미흡**
- TFT 는 **해석가능성 + 의료 도메인 검증 논문 다수** (말라리아·뎅기·독감)
- B2G 납품은 "왜 경보 냈나요?" 답변 가능성이 정확도만큼 중요

우리 프로젝트 적용:
- 입력: 과거 24주 × 3-Layer + 기온·습도
- 출력: 7/14/21일 뒤 위험도 (95% 신뢰구간 병기)

## Slide 7 — 핵심 기술 ②: RAG + 실측 성능

### RAG 경보 리포트
```
경보 발생 → Qdrant에서 유사 과거 사례 검색 → GPT-4o/Claude 리포트 작성
```
> "강남구 Level 4. 유사 사례: 2024년 12월 1주차. 권고: 백신 접종·재고 확보·방역 강화"

### 📊 재현 가능한 실측 성능 (`analysis/notebooks/performance_measurement.py`)
2024-25 시즌 52주 합성 데이터 (seed=42, 재현 가능) 기준:

| 모델 | F1 | MCC | AUPRC | FP |
|---|---:|---:|---:|---:|
| L1 약국 단독 | 0.621 | 0.399 | 0.820 | 11 |
| L2 하수 단독 | 0.615 | 0.359 | 0.838 | 9 |
| L3 검색 단독 | 0.737 | 0.588 | 0.851 | 3 |
| **3-Layer 앙상블** | **0.643** | **0.442** | **0.885** | 10 |

**Granger 인과검정** (statsmodels, maxlag=4): L1 p=0.0000 · L2 p=0.0000 · L3 p=0.0193 → **3 Layer 모두 p<0.05 유의**.

## Slide 8 — 솔직한 자기 진단 (중간발표 핵심 메시지)

### 이전 주장 → 정정
| 구분 | 공모전 시점 | 현재 (정직) |
|---|---|---|
| F1 | 0.71 (하드코딩) | **0.643 실측** |
| Precision | 1.00 단독 강조 | **MCC 0.442 + AUPRC 0.885 병기** (Precision 단독 오해 위험) |
| Granger | p<0.05 HTML 텍스트 | **실측 p-value 표시** |
| 데이터 | "실데이터 26주" | "**재현 가능한 합성** · KDCA 실데이터 수집 진행 중" |

### 왜 수치가 낮아졌나?
공모전 수치는 임계값 최적화 후 가장 좋은 결과만 표기. 현재는 walk-forward CV + MCC·AUPRC 병기 = **더 엄격한 평가 기준** 사용.

### 왜 이게 더 중요한가?
"공모전 수치를 지키는 것"보다 "**공공기관 납품 시 재현 가능한 숫자를 내놓는 것**" 이 B2G 신뢰의 본질.

## 부록 A — 용어 풀이

- **RAG (Retrieval-Augmented Generation)**: 검색 기반 AI 답변 기법
- **TFT (Temporal Fusion Transformer)**: 시계열 예측 AI 모델
- **MCC (Matthews Correlation Coefficient)**: 이진분류 통합 성능 지표 (클래스 불균형 강함)
- **AUPRC**: Precision-Recall 곡선 아래 면적
- **Granger Causality**: 시계열 A가 B를 통계적으로 선행하는지 검정
- **Kafka**: 실시간 데이터 전달 메시지 큐
- **ISMS-P**: 정보보호·개인정보보호 관리체계 인증
- **SaMD**: 의료기기 해당 소프트웨어 (현재 우리 시스템 비해당)
- **WCAG AA**: 웹 접근성 4.5:1 대비율 기준
